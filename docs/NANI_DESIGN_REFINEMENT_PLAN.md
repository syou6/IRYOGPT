# Nani!? デザイン改善計画

## 1. 目的と背景
- 公式サイト・ユーザーレビュー・catnose氏インタビューから得られた最新の知見を整理し、Nani!? のUIをどこまで伸ばせるかを明文化する。
- 展開済みのビジュアルを壊さずに、配色・レイアウト・タイポグラフィ・UIモーションを再設計する際の判断材料を作る。

## 2. リサーチサマリー
### 2.1 全体印象
- 「シンプルで美しい」「余計な機能がなく使い始めやすい」という初見ユーザーの声が多い。[1]
- DotBack記事でも「欲しい情報がトップにまとまっており、翻訳を実行すれば確実に求める出力へ到達できる構成」と評価されている。[2]
- 個人開発者であるcatnose氏は「使いやすさ・気持ちよさを同時に考えながらデザインと実装を進める」ことを重視しており、違和感のないモーション設計が軸になっている。[3]

### 2.2 配色とビジュアル
- メインカラーは淡いウォームブルー（例: `#ecf6ff`）で、背景に薄いグラデーションを敷いて清涼感を演出している。[4]
- ボタンやアイコンにはやや濃い青/薄紫、差し色として黄色やピンクのポップな色を少量使う。
- ニュートラルカラーは#666〜#999のグレー。ホワイトスペースを広くとり、境界線を最小限に抑えている。
- マスコットや3Dライクなイラスト・絵文字風アイコンを多用し、堅さを和らげている。
- 今回のリデザインではNani!?の構造・余白・モーションだけを参照し、色味そのものは既存の`premium.*`パレット（深いグリーン×ブラック×アクセントグリーン）を維持する。

### 2.3 タイポグラフィ
- 丸みのあるサンセリフ（Inter / Noto Sans Rounded系）を想起させるフォントで英日双方を読みやすく表示。
- 見出しは太字・大サイズ、本文はレギュラーウェイトで行間を広くとり、視認性を確保。
- アイコンやラベルは線の太さを揃え、余白でグルーピングする傾向。

### 2.4 レイアウト
- 左サイドバー（約300px）にロゴ、言語切替、主要ナビ（AIほんやく/文章作成）とCTA（無料ではじめる、DLリンク）を集約。
- メインエリアは中央に訳文入力ボックスと送信ボタン、直下に4枚の機能カードを2×2で並べ、情報の優先度を可視化。
- スクロールするとデスクトップ版紹介→対応言語→FAQ→再度CTAと続く。レスポンシブではサイドバーがハンバーガー化し、カードが縦積みになる。[2][4]

### 2.5 ユーザー体験
- 入出力切替アニメーションが滑らかで、読み込み待ちも違和感が少ないとのレビュー。[1]
- FAQやサイドバーで「翻訳データは端末にしか保存されない」「学習に用いない」等の安心材料を繰り返し提示し、不安を打ち消している。
- Mac/Windows 版でも同系色・コンポーネントを共有し、ブランド一貫性を保つ。

## 3. 課題とチャンス
1. **アクセントカラーの一貫性**: 暖色系差し色の使用ルールが明文化されておらず、画面によって印象が散る恐れ。
2. **情報密度の最適化**: 2×2カード直下に説明が続くため、モバイルではスクロール量が多く離脱要因になり得る。
3. **多言語タイポの最適化**: 角丸サンセリフは可読性が高いが、長文説明（FAQ）では字面がやや間延びする。
4. **安心感表現の再配置**: プライバシー説明がサイドバー/FAQに分散しており、ファーストビューでもう一押しの安心訴求が欲しい。
5. **マイクロモーション設計のドキュメント不足**: catnose氏の暗黙知に依存しているため、実装者が変わると体験品質がぶれやすい。

## 4. 改善ガイドライン
### 4.1 配色
- Tailwindの`premium`テーマで定義済みのカラー値（例: `base #040607`/`surface #0B1410`/`elevated #131F1A`/`accent #19C37D`/`accentGlow #7AF4C1`/`warning #F3B664`）をそのまま使用し、新しい色コードは導入しない。
- Nani!?の柔らかさは色ではなくレイヤリングで再現する。`premium.surface`の上に`accentGlow`を5〜8%の不透明度で重ね、角丸+ブラーで淡いハイライトを作り、カード単位で空気感を出す。
- CTAとキーボタンは`premium.accent`を100%でベタ塗り、ホバーで`accentGlow`を外枠に追加する。情報強調に暖色を使いたい場合は既存の`premium.warning`を1:4の比率で差し込む。
- ダークモード/ライトモードの切り替えは既存のトークンに従い、コントラスト比4.5:1以上を維持しているかをチェックリストに追加する。

### 4.2 レイアウト
- モバイル幅ではサイドバーCTAをヘッダー下固定に切り出し、ファーストビューで「無料で翻訳」を常時表示。
- 4枚の機能カードをカルーセル化（3枚表示、スワイプ可能）し、縦スクロールの冗長さを削減。
- FAQは2カラム（デスクトップ）→アコーディオン（モバイル）を維持しつつ、プライバシー系質問を最上位に昇格。

### 4.3 タイポグラフィ
- ヘッダー: 32px/40px, Bold; サブヘッド: 24px/32px, Semibold; 本文: 16px/26px, Regular; キャプション: 14px/22px。
- FAQ等の長文には角丸フォントと合わせて、可読性が高いNoto Sans JP Regularをフォールバックに設定。
- CTAやハイライト文には文字間隔`tracking: 0.02em`を適用し、読み上げ時の視認性を維持。

### 4.4 イラスト・アイコン
- マスコット/3Dイラストの影色は`rgba(0,0,0,0.08)`固定で統一。
- 線画アイコン太さを1.5px、角丸半径8pxに統一。背景シェイプは`premium.elevated`と`premium.warning`を6:4で重ね、`premium.accentGlow`をぼかして柔らかさを出す。

### 4.5 UI/UX & マイクロモーション
- 入力→翻訳→解説表示の各ステップで、200msトランジション+100msステaggerを適用。読み込み時はプログレスバーではなく「3点バウンス」アニメーションで待機を明示。
- プライバシー説明をファーストビューCTA直下に1行で追加し、詳細ページへリンク（例: "翻訳データは端末にのみ保存されます"）。
- デスクトップ/モバイルで体験差が出やすい部分（カードカルーセル、FAQアニメーション）はStorybookか同等のプレビューで共有する。

## 5. 実装チェックリスト
| カテゴリ | 確認項目 |
| --- | --- |
| 配色 | トークン化されたカラーがFigma/コード双方で同期されているか |
| レイアウト | 768px/1024px/1280pxで主要CTAがファーストビューに残っているか |
| タイポ | 日本語・英語混在文で改行/行間が破綻していないか |
| アイコン | 線幅・角丸・配色ルールが一貫しているか |
| アニメーション | 300ms以内で完了し、Reduce Motion設定時はフェードにフォールバックしているか |
| 安心表示 | プライバシー説明がサイドバーとファーストビューの両方に存在するか |

## 6. UIレイアウト仕様
### 6.1 ページ全体
- コンテナ幅は最大1200px（`max-w-6xl`相当）で左右24pxパディング。背景には`bg-premium-grid`と`bg-premium-radial`を重ね、Nani!?の淡い光を既存パレットで再演。
- レイアウトは `flex` で左サイドバー + 右メイン。ブレークポイントは`lg=1024px`で2カラム、それ未満ではサイドバーをページ上部に展開し、CTAのみをヘッダー下に固定。
- セクション間隔は64px、Surface/Cardは`p-6（sm:p-8）`で統一。角丸は`Surface=40px`、カード=28pxとして丸みを強調。

### 6.2 サイドバー
- 幅320px固定。構成: (1) ロゴ + ラベル (2) 製品説明テキスト (3) ナビリンク（`AIほんやく/特徴/デスクトップ版/FAQ`） (4) 安心コピー3本 (5) プライマリCTA「無料で翻訳してみる」(6) セカンダリCTA「プランを比較」(7) プライバシー追記。
- `position: sticky; top:24px`で常に表示。モバイルでは各ブロックをアコーディオン化し、CTAのみ常設。
- ナビは`rounded-3xl`ボタンに矢印とホバー時の`premium.accent`ラインを付加。安心コピーは`premium.elevated`背景に`accentGlow`を5%で重ね柔らかさを出す。

### 6.3 ヒーロー翻訳デモ
- 右メインの冒頭Surface。上段に見出し/リード文、その下に翻訳デモカードを配置。カード内は左列=入力（原文、トーンラベル、コピーアクション）、右列=出力（訳文、解説、バッジ、バリエーションチップ）。
- トーン切替はラベルチップ（カジュアル/ビジネス/クリエイティブなど）を水平スクロール可能に配置。選択状態は`border-premium-accent`＋`bg-premium-elevated`で表現。
- 出力カードには`shadow-glow`を適用し、Nani!?のハイライト感を再現。ローディング時は3点バウンスアニメーションをカード下部に挿入。

### 6.4 情報カード群
- ヒーロー直下に2×2の機能カード（`FEATURE_CARDS`）。各カードはアイコン→タイトル→説明の縦並びで余白24px。モバイルでは縦積みで`gap-3`。
- 続くステップカードは3列（STEP 01〜03）で、Nani!?同様に番号バッジと本文を余白大きめに配置。
- デバイス紹介カードは2列。各カード上部にセクションラベルを置き、Bulletは`●`+テキストで行間16px。

### 6.5 FAQ / CTAセクション
- FAQはフラットなアコーディオン。`button`全体をクリック領域にし、開閉時に高さがスムーズに変わるよう`max-height + transition`を適用。最初の質問にプライバシー関連をセット。
- 最終CTAは「今すぐ試す」セクションとして、CTAポイント（ログイン不要/端末保存/AIは学習しない）のピル型タグを並べ、下にプライマリ＋セカンダリボタンを横並びに配置。

### 6.6 スクロール順/ナビゲーション
1. サイドバー（常設）
2. ヒーロー翻訳デモ + カード
3. 指標/ステップ/デバイス紹介
4. FAQ
5. 最終CTA
- 各セクションに`id`を付与し、ナビリンクと一致させる。スクロール遷移は`scroll-margin-top`を24pxに設定し、Nani!?同様スムーズスクロールする。
