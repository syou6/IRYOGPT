# 入社１か月で、同期が私から孤立した話

「内定おめでとう！」

秋の気配が漂う頃になると、SNSや街の雑談からそんな声が聞こえてくる。

SNSを開けば、満面の笑みを浮かべた内定式の写真がタイムラインを流れていく。スーツに身を包み、未来を誇らしげに語る同世代の姿を見て、つい一年前の自分を思い出す。

私がSIerに新入社員として入社したのは、たった一年半前のことだ。私は新卒でSIerに入社し、「ここから長い会社人生が始まるのだ」と、どこか浮き立つ気持ちでいた。

同期は友達とは少し違うけれど、同じ時期に同じスタートを切った仲間。これから切磋琢磨して成長していける、そんな風に信じていた。

けれど現実は思ったよりも冷たく、入社からわずか一か月で、同期は「孤立」という予想もしなかった状況に立たされることになる。いったい何が起きたのだろうか。

## ちやほやされた自己紹介プレゼン

入社してすぐの研修期間、新入社員は一人ずつ「自己紹介プレゼン」をすることになった。与えられた数分の時間で、自分がどんな人間で、これまで何をしてきたのかを伝える。

私はそこで学生時代の研究成果を発表した。学会発表の経験、受賞歴、プログラミング…。これまで努力してきた（？）軌跡を淡々と語っただけだ。

大学時代から各所で評価され、周りからは将来を期待されており、周りから少し特別扱いを受けてきた――そんな過去があったからこそ、自然とその延長線上で自分を紹介した。

それでも発表を終えると、周囲の空気が一変した。

「すごいね」「めっちゃ優秀じゃん」

――羨望や尊敬の眼差しが一斉に向けられた。学生時代と同じように、再びちやほやされる心地よさを感じた。「あ、ここでも上手くやっていけそうだな」とその瞬間思った。

## 悪平等にイライラとして

最初の数日は特別視されていた私も、やがて扱いは「同期の一人」に戻っていった。会社という場所は、実力や成果よりも「同じ新人」という括りで人を見ようとするのだろう。

だがそこで摩擦が生まれた。私以外の同期の多くは、プログラミング初心者だった。研修課題でようやくHello Worldを動かすレベルで、ちょっとしたコードも書けない人もいた。

上司や講師もまた、私を特別扱いすることはなかった。課題を早く終えても、「手が空いたなら、みんなをサポートしてあげて」と言われる。褒められるわけでもなく、ただ「新人だから当然」といった空気で処理される。

頭では理解している。会社は個人戦ではなく、チームワークが求められる場なのだと。けれど、努力して積み重ねてきた差が無視され、同じ一括りにされることに、苛立ちを覚えずにはいられなかった。

## 群れる同期と孤立する私

そんな違和感を抱えたまま日々が過ぎ、同期同士は自然と群れるようになった。お昼ごはんに連れ立って行く、帰りに雑談をしながら歩く、週末に遊びに行く…。

だが私はその輪に加わるのが苦痛だった。話題はたいてい、誰が誰と付き合っていたとか、学生時代のくだらない失敗談とか、私にとっては興味のない話ばかり。そんな「生産性のない会話」に時間を費やすのが耐えられなかった。

「この時間で論文を1本書いた方がいいんじゃない？」
「デザインパターンを１つ覚えたほうが将来につながるのに」

そんな風に思ってしまい、気づけば一人で行動することが多くなった。最初は不思議そうにしていた同期も、次第に何も言わなくなった。私が単独行動するのも自然な流れだったのだろう。

## 「群れない自由」という答え

こうして振り返ると、私が同期から孤立したのは偶然ではなく、必然だったのかもしれない。私は自分の時間を大切にし、生産性を求め、無駄を嫌った。対して同期たちは、人間関係を築くことそのものに価値を置いていた。

どちらが正しいという話ではない。ただ価値観の違いが、距離を生んだのだ。私はその後、会社での退屈な日々が嫌になり、朝から晩まで、本のストーリーを考える日々を迎えていく。本に没頭することで、１年後、ベストセラーになることは、当時は知るよしもなかったけれど。

「内定取った」と喜ぶ声を聞きながら、私は思う。

これから社会に出る人たちに伝えたいのは、「群れることが正義ではない」ということだ。孤立は悪いことではない。自分の軸を持ち、何を優先するかを選び取る勇気の表れでもある。

一年半前の私は、その選択をした。孤立の代わりに得た時間と集中力が、今の私をつくっている。
