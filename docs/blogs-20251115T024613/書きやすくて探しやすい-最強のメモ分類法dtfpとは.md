# 書きやすくて探しやすい! 最強のメモ分類法DTFPとは?

「アプリを乗り換えても、ノートを変えても、メモが続かない。」

メモを取ると役に立つと言う人はたくさんいるけれど、いざなんとなく書いてみても、情報が散らばり、必要なときに見つからず、やがて記録する気力も失せてしまう。

──多くの人がそう感じる理由は、書き方が悪いのではなく、分類の仕方が悪いからだ。思いついたことをとりあえず書き留めても、後で探すときに見つからなければ意味がない。逆に、整理しすぎると書くのが億劫になる。

結局のところ、メモは「書く」と「探す」という二つの行為の間で綱引きをしている。どちらかに偏れば、もう一方が犠牲になる。多くのメモ術は、このバランスの設計を置き去りにしている。

私が３年ほどの試行錯誤の末にたどり着いた「DTFP分類法」は、その両立を自然に実現した。プロンプト、メール作成、プレゼン制作、設計書ーーどんな成果物を作る際も使える、AI時代に欠かせない方法を紹介したい。

## Daily memo ― 毎日の出来事と気付き

Daily memoは、1日ごとに書くメモだ。今日あったこと、感じたこと、ちょっとした気付き。これらを淡々と日付ごとに残している。ポイントは、**日付で区切る**こと。

人間の記憶は、意外と出来事の順番や場所よりも「日付」で呼び出しやすい。たとえば、「あの時何を考えていたか」は忘れていても、「その日何をしていたか」は覚えていることが多い。だから、カレンダーの予定を見返せば、当時の出来事や思考のきっかけを探し出せる。

Daily memoは、書く時点では特に整理しない。思いついたことをそのまま書き散らす。それでも日付というラベルがあるおかげで、後から必要な情報を見つけるのは意外と簡単だ。情報管理に凝りすぎると、書くのが面倒になり、結果として書かなくなる。このやり方は、そのちょうどいい塩梅を保ってくれる。

## Temp memo ― 短期的な作業用

Temp memoは「臨時メモ」。何かを作ったり考えたりする、その作業のためだけに使う。たとえば、ブログ記事の構成を練るとき、プレゼンのスライド案を並べるとき、ソフトウェアの設計図を描くときなど。

この分類の役割は明確で、**プロジェクトの下書き置き場**だ。必要がなくなれば消してもいいし、そこからFormal memoやPublic documentsに昇格させてもいい。重要なのは、「ここは一時的なスペース」という前提があること。そうすると、内容が荒くても、箇条書きでも、気楽に書ける。100均で買ったノートに自由に書いて、使い終わったら捨てたり、スマホやPCにメモをしたりする。

私の場合、Temp memoは毎日かなりの回転率で入れ替わっていく。形になったものは別の場所へ、不要になったものは削除。おかげで「メモが溜まりすぎて管理不能」という状態にならない。

## Formal memo ― 仕事と知識の土台

Formal memoは、用途別に整理された正式なメモ。自分の歴史記録、計画書、Zettelkastenのメモなどがここに入る。外部に出すことを前提とせず、あくまで自分の仕事や思考を支えるための文書だ。

分類は日付でも用途でもよい。たとえば、進行中のプロジェクトなら日付順に経過を残す。一方で、知識を積み上げていくタイプのメモはテーマ別に分類する。Zettelkastenのような構造的な整理はここで行う。

Daily memoやTemp memoとの大きな違いは、「残す前提で書く」ことだ。ここに入る文章は、将来の自分が確実に使うことを見越している。だから、文章も図も、後から見て意味が分かるように整えておく。

## Public documents ― 外に出す文章

最後がPublic documents。これはブログ記事、公式サイトの文章、SNS投稿など、外部に出すことを前提とした文書だ。厳密には「メモ」ではないかもしれないけれど、DTFPに入れている理由は、**内部と外部を明確に区別するため**だ。

ここに入れるのは、自分の考えを外部に表明するための文章だ。DailyやFormalから育ったものを清書して持ってくることも多い。逆に、外に出す予定のない文章は絶対にここには入れない。この境界があるおかげで、頭の中のアイデアと世間に出す言葉を混同せずに済む。

Public documentsは、最終的に誰かの目に触れる。だから、文章の質や見せ方を意識するようになる。内部情報と外部情報の棲み分けを自然に考えるようになる。だから結果的に、自分の書く力を鍛える場所にもなっている。

## アイデアを枯らさない方法

メモの価値は、「書きやすさ」と「引き出しやすさ」の両輪で成り立っている。どちらか片方だけでは不完全だ。文房具メーカーが追求する滑らかな書き心地も、メモマニアが追い求める整理術も、それ単体では実用に耐えない。

メモの本質は、日々の思考と情報をスムーズに循環させることだ。DTFPは、記録と活用の両面を無理なく満たす構造を持っている。無理にきれいに整える必要も、完璧に分類する必要もない。

重要なのは、毎日使えて、必要なときに必ず役立つことだ。
