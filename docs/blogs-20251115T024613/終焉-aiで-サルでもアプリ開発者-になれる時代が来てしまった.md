# 【終焉】AIで“サルでもアプリ開発者”になれる時代が来てしまった

「プログラマが失業する」

――そんな言葉を聞いても、冗談か煽り文句のように思う人は多いだろう。
実際、エンジニアの多くは「自分の仕事が代替されるのはまだ先」と考えている。

しかし現実は、想像以上に早く迫ってきている。

少し前まで、プログラミングは専門知識と経験を要する高度な技能だった。
数千行のコードを設計し、手作業で書き上げる。そこには職人技のような繊細さがあり、「誰にでもできるものではない」という認識が社会に根付いていた。

だからこそ、士業に並ぶ高給と安定が保証されていたのだ。

だがAIの登場は、その構図を一変させる。
「ホームページを作って」「このデータを分析して」と指示するだけで、数分後には動くプログラムが生成される。しかも実用に耐える完成度を持って、人間が何日もかけて積み上げた作業を、AIが一瞬で肩代わりしてしまうのだ。

なぜ、プログラマが失業すると言われているのか。それは、簡単な作業であれば、もはやサルでもできるといえるくらい容易にこなせてしまう現実が、すでに目の前にあるからだ。

## Vibe Codingの広がり

昔のエンジニアは、自分でコードを考えてゼロから書いていた。仕様を理解し、構造を設計し、処理の流れを緻密に組み立てていく。そこには「創造」や「設計」という知的な営みがあった。結果的に、この仕事は高度な訓練を受けた一握りの人間のものだった。

ところが今は、「Vibe Coding」という新しいスタイルが広がりつつある。これは、完成形を細かく設計するのではなく、「こんな雰囲気のアプリが欲しい」とざっくり指示を出し、その“雰囲気”に沿ってAIに作らせるやり方だ。

デザイン、機能、構造のすべてがプロンプトの文章から自動生成されるため、開発の出発点はアイデアの温度感や方向性で決まる。細かい仕様は後から調整すればよく、初期段階では“とりあえず動くもの”がすぐに手に入る。

これは、プログラミングの常識を大きく変えてしまった。以前は、コーディングするにも、膨大な時間がかかっていた。一ヶ月に1000行のプログラムをかければ、会社から給料を貰えていた。しかし、今はその程度のプログラムであれば、適切な設計書さえあれば、つまり、かつてプログラマに出していた指示書さえ作れば、作れるようになってしまった。

そのため、コードを書くこと自体に価値はほとんどなくなってしまった。

（ただし、設計書を書くことは、人とのコミュニケーションや業務フローの観察が必要なためまだ完全には代替できず、こちらの仕事は当面はなくならないと思われる。）

## 成果は当たり外れの“ガチャ”

とはいえ、このAI時代のプログラミングには特徴的な制約がある。それは「結果が当たりか外れか、やってみるまでわからない」ということだ。

たとえば、1万行程度のプログラムなら、AIは驚くほど簡単に作ってくれる。しかしその完成度は6割ほど。動作はするものの、細かい仕様や例外処理までは手が届かないことも多い。さらに、外部サービスとの接続や、セキュリティを考慮した実装は、まだ人間が微調整しなければならない。

とはいえ、簡単な業務ツールや趣味のアプリなら、この方法であっという間に作れる。内部的には欠陥だらけでも、機能を絞り、自分だけで使う分には、なんの問題もない。

（しかし中には、この未完成品をそのままアプリストアやWebで公開してしまい、大きなセキュリティ事故やデータ漏洩事件につながるケースもある。）

## わずか8時間でHPを公開する方法

では、実際にAIを使って私がどのようにHPを作ったのかを紹介しよう。

-

**Manusに指示を出す**
　まず「HPを作って」と要望を投げる。すると、Manusが複数のデザイン案を提案してくれる。10パターンほど生成させると、その中に「これだ」と思えるものが必ず出てくる。

-

**気に入ったものをzipでダウンロード**
　完成度の高い案を選び、zipファイルとしてダウンロードする。これだけで一応のWebサイトは手元に揃う。

-

**Claude Codeで修正**
　ダウンロードしたソースをClaude Codeに読み込ませ、細部を修正していく。レイアウトの崩れやテキストの微調整は、AIに頼めばすぐに直せる。

-

**Vercelにデプロイ**
　修正したサイトをVercelにデプロイすれば、公開完了だ。

この流れを踏むことで、わずか8時間ほどで下記のようなHPが完成する。

👉 [https://www.hundreds.co.jp/](https://www.hundreds.co.jp/)

もちろん、Next.jsをVercelにデプロイできる程度の知識は必要だが、これは10時間ほど勉強すれば習得できる範囲だ。普通のプログラマなら一晩で追いつけるレベルであり、AI時代の「Web制作」のハードルは想像以上に低い。

（スマホアプリもClaude Codeで開発できるが、もう少し複雑。ただ、情報工学を大学院まで6年間真面目に勉強してきた有名大学卒の人なら誰でも1日で簡単なアプリ程度なら作れる程度の難易度。）

## ボタンを押すだけでアプリができる！

ここまでを振り返ると、Vibe Codingは心理学の「スキナー箱実験」と本質的に同じ構造だと気づく。

スキナー箱とは、レバーを押すと餌が出る仕組みの実験装置で、動物は「レバーを押せば報酬が得られる」と学習する。だが、動物自身はレバーや機械の内部構造を理解しているわけではない。単に「押す＝もらえる」という因果関係だけを利用しているのだ。

AI時代のプログラミングも同じだ。私たちはコードの細部を理解せずとも、「作って」と入力すればアプリやWebサイトが出てくる。内部の仕組みや複雑なアルゴリズムを知らなくても、報酬（＝完成品）を得られる。違いは、猿はバナナをもらい、私たちはアプリをもらうだけだ。

今は1万行程度のプログラムを作るのに一苦労する段階であっても、来年には1万行が10万行になり、100万行できるようになる可能性だってある（一般的な個人開発のスマホアプリは数十万行程度。大企業が作っているアプリだと数百万行)。つまり、大規模アプリでさえAIが作る時代はすぐそこにある。

結局のところ、「プログラマが失業する」という言葉は脅しでも冗談でもない。5年後か10年後かは分からないけれど、電話交換手と同じで、いずれこの仕事はなくなるだろう。

「AIに仕事を奪われる」と笑っていられるのは今のうちだ。数年後には、あなたの仕事が“奪われる側”になっているかもしれない。
